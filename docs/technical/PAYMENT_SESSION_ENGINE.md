# Payment Session Engine

Payment Session v0 is the planned coordination layer for paid checks, reviews, and future guarded flows.

Session data:
- user or Telegram identity
- selected product
- chain, token, amount, receiver wallet, and optional agent
- payment provider
- invoice
- payment proof or provider reference
- entitlement
- audit log

The MVP uses manual payment proof submission. Full auto-unlock is planned but must be auditable.
