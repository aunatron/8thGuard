# Escrow And Custody Gates

8thGuard does not provide production escrow or custody in the MVP.

Before any protected transaction, custody-like, or escrow-like feature can launch:
- counsel must review legal status and user disclosures
- security must review transaction architecture
- custody boundaries must be documented
- dispute resolution must be documented
- audit logs must be persistent
- incident response must be ready
- the product must not request private keys or seed phrases

Until then, all guarded transaction messaging must say future, gated, and not live.
