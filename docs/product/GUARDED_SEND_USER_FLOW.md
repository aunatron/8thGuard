# Guarded Send User Flow

1. User chooses Guarded Send.
2. User enters chain, token, amount, receiver wallet, and optional agent.
3. 8thGuard runs wallet and context checks.
4. 8thGuard quotes service/protection fees.
5. User pays through official Paystack or crypto service-payment rail.
6. Payment session creates audit log, invoice, ledger entry, and entitlement.
7. User receives risk indicators and next-step guidance.
8. Future protected transaction execution remains gated until legal, security, and compliance review.

Today this flow is informational and architecture-ready only.
